## 4. Рекомендация

### 🏆 Первый домен: Логистика / ВЭД

#### Почему логистика — правильный выбор для MVP:

**1. Уникальная комбинация факторов**
- Единственный домен, где есть И domain knowledge, И готовая кодовая база, И пилотный клиент
- Никто другой не может повторить это преимущество

**2. Логистический домен максимально нагружает Aether**
- Channels: Telegram-боты, Web-портал, API (1C, ЭТРАН, ATI.SU)
- AI Core: классификация заявок, парсинг документов (NLP), расчёт стоимости, оценка рисков
- Services: заявки, документы, отслеживание, расчёт стоимости — все 4 pillar Aether
- Delivery: уведомления, статусы, webhook-интеграции

**3. Реальный бизнес-кейс (МТК)**
- МТК может стать enterprise-пилотом Aether
- Реальные данные → реальные метрики → реальный кейс для продаж
- Нет необходимости «изобретать» use-кейсы — они уже работают в Logicore

**4. Конкурентная дыра**
- Пространство «AI-воронка для ВЭД/логистики» практически свободно
- 1С:Логистика — устаревшая, без AI
- Битрикс24 — generic, без предметной логики
- Ниша Дальнего Востока / контейнерных перевозок — практически монопольна

**5. Reuse для других доменов**
- Логистика — самый «тяжёлый» домен по интеграциям и services
- Если Aether работает для логистики — он будет работать для любого B2B-домена
- Абстракции, созданные для логистики (документы, статусы, каналы), напрямую переносятся

#### План MVP (3-4 месяца)

| Месяц | Фаза | Дедлайны |
|-------|------|----------|
| **Месяц 1** | Foundation | Multi-tenant архитектура, Channels (Telegram + Web), AI Core (классификация заявок) |
| **Месяц 2** | Services | Документооборот, расчёт стоимости, трекинг, RBAC |
| **Месяц 3** | Integration | 1C API, ЭТРАН, ATI.SU, White-label конфигурация |
| **Месяц 4** | Pilot | Onboarding МТК, QA, метрики, итерации |

#### Стратегия выхода за пределы MVP

1. **Месяц 4-6:** Второй клиент из логистики (экспедитор или таможенный брокер)
2. **Месяц 6-9:** B2B услуги (юристы/бухгалтеры) — reuse AI Core + Channels, замена Services-слоя
3. **Месяц 9-12:** E-commerce — расширение Channels, новые Services

## 6. Ключевые выводы

1. **Логистика/ВЭД — единственный домен со score > 8.** Все остальные — 3-6.
2. **Личная экспертиза — решающий фактор.** Без неё даже лучший домен по TAM будет дорогим MVP с высоким risk.
3. **Logicore = free R&D.** 4 года разработки, 130K+ строк, 7 AI-агентов — это бесплатно пройденный путь для Aether.
4. **Рекомендуемый путь:** Логистика (MVP) → B2B Услуги (Stage 2) → E-commerce (Stage 3) → Медицина (Stage 4, если вообще).
5. **Медицина — не MVP.** Слишком высокая регуляторика, сильная конкуренция, нет экспертизы. Может быть Stage 4+ после PMF в других доменах.
